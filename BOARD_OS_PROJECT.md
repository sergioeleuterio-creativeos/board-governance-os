# Board Governance OS

AI governance layer for founder-led companies.

This workspace is seeded from the working CreativeOS Next/Supabase application, then narrowed to the Board Governance OS production loop:

Company Brain -> Governance Run -> Board Pack -> Shadow Board Review -> Board Meeting Session -> Decision Memory -> Follow-ups -> Founder Dashboard.

Positioning guardrail: Board Governance OS is not an AI board member, board replacement, or virtual CEO. It is a governance operating system for better founder decisions.

Project memory lives in `docs/SHADOW_BOARD_MEMORY.md`.

Production foundation lives in `docs/PRODUCTION_FOUNDATION.md`.

Full sprint plan lives in `docs/SPRINT_PLAN.md`.

Daily routines:
- `/startday`: read memory, inspect current state, and propose the plan of the day from `docs/STARTDAY_ROUTINE.md`.
- `/endday`: close the sprint/day by updating memory, pending items, backlog, and bugs from `docs/ENDDAY_ROUTINE.md`.
